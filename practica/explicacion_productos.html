1. Que pide la consigna

Punto 1 - Consigna textual
"Una distribuidora de articulos de libreria y papeleria dispone de un archivo con los pedidos realizados a lo largo del anio 2019. El archivo tiene formato CSV (valores delimitados por punto y coma). Desglosando, el programa debe:

  • Leer el archivo pedidos.csv con formato CSV separado por punto y coma

  • Para cada articulo, determinar el mes con mayor demanda
  • Mostrar el listado ordenado alfabeticamente por descripcion
  • Cada articulo aparece una sola vez
  • Crear una funcion recursiva para contar elementos de un conjunto

Requisitos tecnicos:

  • Usar manejo de excepciones
  • Crear funciones que modularicen el codigo
  • Incluir al menos una funcion lambda
  • Comentar la estructura de datos elegida y justificar

2. Formato del archivo pedidos.csv

Cada linea del archivo tiene 4 campos separados por punto y coma (;):

Campo Descripcion Ejemplo
SKU Codigo unico del articulo 12316
Descripcion Nombre del articulo Broche Dorado N8 * 100
Fecha Fecha del pedido (DD/MM/AA) 17/12/19
Cantidad Unidades solicitadas 85

Ejemplo de una linea real del archivo:

12316;Broche Dorado N8 * 100 3.3 Cm;17/12/19;85

12316 = SKU (codigo del articulo)
Broche Dorado N8... = Descripcion del articulo
17/12/19 = dia 17, mes 12 (diciembre), anio 2019
85 = 85 unidades pedidas
Clave: A diferencia del previo 2021 que usaba formato fijo (slicing), aca se usa split(";") para separar los campos y split("/") para separar la fecha. Mucho mas legible.

3. Estructura de datos elegida

Elegimos un diccionario con listas y sub-diccionarios:

articulos = {
    "12316": [ "Broche Dorado N8...", { "12": 85, "05": 200 } ],
    "5332":  [ "Abro.Maped 26/6...",  { "01": 109, "03": 340 } ],
    # ... miles de artículos más
}

# Clave externa: SKU del artículo
# Valor: [descripcion, {mes: cantidad_total_acumulada}]

Por que esta estructura y no otra?

Estructura Acceso Problema
Diccionario (elegida) O(1) por SKU Ninguno: acceso directo para buscar y acumular
Lista O(n) secuencial Con 50000 registros, buscar un SKU recorriendo toda la lista es muy lento
Lista enlazada O(n) secuencial Mismo problema, mas complejidad de implementacion
Arbol binario O(log n) Innecesario: no necesitamos orden especial para busqueda

El sub-diccionario de meses dentro de cada articulo permite acumular cantidades por mes de forma eficiente. Al final, basta con buscar el mes con valor maximo.

5. Funcion leer_archivo()

Cumple con
Lectura del archivo CSV + acumulacion por articulo y mes + manejo de excepciones + modularizacion 
def leer_archivo():
    """Lee pedidos.csv y devuelve un diccionario con las cantidades por mes de cada artículo."""
    articulos = {}
    try:
        archivo = open("pedidos.csv", "r")
        for linea in archivo:
            linea = linea.strip("\n")
            # Formato CSV: SKU;descripcion;DD/MM/AA;cantidad
            SKU, descripcion, fecha, cantidad = linea.split(";")
            dia, mes, anio = fecha.split("/")
            cantidad = int(cantidad)

            # Si el artículo es nuevo, creo su entrada con descripcion y dict de meses
            if SKU not in articulos:
                articulos[SKU] = [descripcion, {}]
            # Si el mes es nuevo para este artículo, lo inicializo en 0
            if mes not in articulos[SKU][1]:
                articulos[SKU][1][mes] = 0
            # Acumulo la cantidad vendida en ese mes
            articulos[SKU][1][mes] += cantidad
    except FileNotFoundError:
        print("Error: no se encontró el archivo pedidos.csv")
    except OSError:
        print("Error: problema al leer el archivo")
    finally:
        try:
            archivo.close()
        except NameError:
            pass
    return articulos

6. Funcion mes_mayor_demanda()

Cumple con
Determinar el mes con mayor demanda + funcion lambda (requisito) 
def mes_mayor_demanda(cantidades_por_mes):
    """Recibe un diccionario {mes: cantidad} y devuelve el mes con mayor cantidad."""
    return max(cantidades_por_mes, key=lambda m: cantidades_por_mes[m])

Como funciona max() con lambda:

# Supongamos: cantidades_por_mes = {"01": 109, "05": 350, "12": 85}

max(cantidades_por_mes, key=lambda m: cantidades_por_mes[m])

# max() recorre las claves: "01", "05", "12"
# Para cada clave, la lambda devuelve su valor:
#   "01" -> 109
#   "05" -> 350    <-- maximo!
#   "12" -> 85
# Resultado: "05" (el mes con mayor cantidad)
Atencion: max() sobre un diccionario recorre las claves, no los valores. La lambda le dice "compara usando el valor asociado a cada clave". Sin la lambda, max() compararia las claves como strings ("12" > "05" > "01") lo cual seria incorrecto.

7. Funcion nombre_mes()

def nombre_mes(num_mes):
    """Convierte un número de mes (string de 2 dígitos) a su nombre en español."""
    meses = {
        "01": "Enero", "02": "Febrero", "03": "Marzo",
        "04": "Abril", "05": "Mayo", "06": "Junio",
        "07": "Julio", "08": "Agosto", "09": "Septiembre",
        "10": "Octubre", "11": "Noviembre", "12": "Diciembre"
    }
    return meses.get(num_mes, num_mes)

Funcion auxiliar que convierte "05" en "Mayo" para que el informe sea mas legible. Usa .get() para que si el mes no existe en el diccionario, devuelva el numero tal cual (nunca deberia pasar, pero es una buena practica defensiva).

def mostrar_informe(articulos):
    """Muestra el listado ordenado alfabéticamente por descripción."""
    if not articulos:
        print("No se encontraron artículos.")
        return

    # Armo una lista de tuplas (descripcion, SKU, mes_mayor, cantidad) para ordenar
    listado = []
    for SKU, datos in articulos.items():
        descripcion = datos[0]
        cantidades = datos[1]
        mejor_mes = mes_mayor_demanda(cantidades)
        listado.append((descripcion, SKU, mejor_mes, cantidades[mejor_mes]))

    # Ordeno alfabéticamente por descripción
    listado.sort(key=lambda x: x[0])

    print(f"{'='*85}")
    print(f"  LISTADO DE ARTÍCULOS - MES DE MAYOR DEMANDA")
    print(f"{'='*85}")
    for desc, sku, mes, cant in listado:
        print(f"  SKU {sku:<8} | {desc:<35} | {nombre_mes(mes):<12} | {cant} unidades")
    print(f"{'='*85}")
    print(f"  Total de artículos: {len(listado)}")

9. Funcion contar_elementos() - Recursiva

Cumple con
"Funcion recursiva que cuente la cantidad de elementos de un conjunto sin utilizar len(), cuyo funcionamiento se base en el manejo de excepciones." 
def contar_elementos(conjunto):
    """Cuenta los elementos de un conjunto sin usar len().
    Usa recursión y manejo de excepciones: cuando el conjunto está vacío,
    pop() lanza KeyError, que funciona como caso base."""
    try:
        conjunto.pop()  # Saco un elemento; si está vacío lanza KeyError
        return 1 + contar_elementos(conjunto)
    except KeyError:
        return 0  # Conjunto vacío: caso base

Como funciona la recursion:

La idea es simple: saco un elemento con pop(), cuento 1, y recursivamente cuento el resto. Cuando el conjunto esta vacio, pop() lanza KeyError, que actua como caso base.

# Ejemplo: conjunto = {10, 20, 30}

contar_elementos({10, 20, 30})
    pop() -> saca 10, queda {20, 30}
    return 1 + contar_elementos({20, 30})
                    pop() -> saca 20, queda {30}
                    return 1 + contar_elementos({30})
                                    pop() -> saca 30, queda {}
                                    return 1 + contar_elementos({})
                                                    pop() -> KeyError!
                                                    return 0
                                    return 1 + 0 = 1
                    return 1 + 1 = 2
    return 1 + 2 = 3   # Correcto!
Importante: pop() modifica el conjunto original, por eso al llamarla usamos contar_elementos(ejemplo.copy()) para no destruir el conjunto.

Por que KeyError y no IndexError?

Los conjuntos (set) en Python usan pop() que lanza KeyError cuando estan vacios (no IndexError como las listas). Esto es porque los conjuntos no tienen indices, solo claves.

Verificacion:

ejemplo = {10, 20, 30, 40, 50}
print(f"Conjunto: {ejemplo}")
print(f"Cantidad (sin len): {contar_elementos(ejemplo.copy())}")  # 5
print(f"Verificación con len(): {len(ejemplo)}")                  # 5
