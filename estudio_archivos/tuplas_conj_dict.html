Tuplas

Que es una tupla?

Las tuplas son similares a las listas: se accede a sus elementos a traves de un subindice. La diferencia principal es que las tuplas son inmutables (no se pueden modificar).

LISTA
[ ] Mutable
vs
TUPLA
( ) Inmutable

Crear tuplas

# Con parentesis
salsas = ("tartara", "criolla", "chimichurri")

# Sin parentesis (tambien crea tupla)
salsas = "tartara", "criolla", "chimichurri"

# Tupla vacia
aderezos = ()

# Tupla de UN solo elemento (coma obligatoria!)
misalsa = ("barbacoa",)
misalsa = "barbacoa",

Sin la coma final, NO es tupla, es solo un string entre parentesis.

Tipos mixtos y anidamiento

# Tipos mixtos
primavera = (21, "Septiembre")

# Tuplas anidadas
alumno = (152408, "Luis Arce", "Lima 717", (1, "Mayo", 1998))
print(alumno[1])      # Luis Arce
print(alumno[3][1])   # Mayo

Inmutabilidad - TypeError

t = (1, 2, 3)
t[0] = 4   # TypeError: 'tuple' object does not support item assignment

Las tuplas NO se pueden modificar por subindice.

Concatenar y repetir

# Concatenacion (crea nueva tupla)
aderezos = ()
aderezos = aderezos + ("mayonesa",)

# Repeticion
binario = (0, 1) * 3
print(binario)  # (0, 1, 0, 1, 0, 1)

Funciones y metodos disponibles

SI Disponibles

  • len(), max(), min(), sum()
  • Operador in / not in
  • .index(), .count()
  • Rebanadas (slicing)
  • Recorrido con for

NO Disponibles

  • .append()
  • .remove()
  • .pop()
  • .sort()
  • Cualquier metodo que modifique in situ

Conversion y aplicaciones

# Lista a Tupla
lista = [1, 2, 3]
tupla = tuple(lista)   # (1, 2, 3)

# Tupla a Lista
nueva = list(tupla)    # [1, 2, 3]

# Intercambio de variables (tuple unpacking)
a, b = b, a

# Uso tipico: datos heterogeneos
fecha = (dia, mes, anio)

Conjuntos (Sets)

Que es un conjunto?

Un conjunto es una coleccion de elementos sin orden y sin duplicados. No es una secuencia (no tiene subindices). Ideal para eliminar repetidos.

frutas = {"banana", "manzana", "naranja", "pera", "banana"}
print(frutas)  # {"banana", "manzana", "naranja", "pera"}
# "banana" quedo solo una vez

Conjunto vacio: set() NO {}

conjunto = set()   # Conjunto vacio CORRECTO
oops = {}           # Esto es un DICCIONARIO vacio, NO un conjunto!

Mutabilidad del conjunto vs elementos

Los conjuntos son mutables (se pueden modificar), pero sus elementos deben ser de tipo inmutable (numeros, strings, tuplas).

# OK: elementos inmutables
validos = {1, "hola", (2, 3)}

# ERROR: listas son mutables, no se pueden agregar
lenguajes = {["Python", "Perl"]}  # TypeError: unhashable type: 'list'

Operaciones de conjuntos

|Union - A U B - Todos los elementos de ambos
&Interseccion - A ∩ B - Solo los comunes
-Diferencia - A - B - Los de A que no estan en B
^Diferencia simetrica - A Δ B - Los que NO son comunes
A = {1, 2, 3, 4}
B = {3, 4, 5, 6}
print(A | B)   # {1, 2, 3, 4, 5, 6}
print(A & B)   # {3, 4}
print(A - B)   # {1, 2}
print(A ^ B)   # {1, 2, 5, 6}

Metodos de conjuntos

add(elem)Agrega un elemento (como append en listas)
remove(elem)Elimina elem. KeyError si no existe
discard(elem)Elimina elem. NO da error si no existe
pop()Elimina y devuelve un elemento al azar. KeyError si vacio
clear()Elimina todos los elementos
conj = {3, 4, 5}
conj.add(6)       # {3, 4, 5, 6}
conj.remove(4)    # {3, 5, 6}
conj.discard(99)  # Sin error, {3, 5, 6}
x = conj.pop()     # Saca uno al azar
conj.clear()       # set()

Diccionarios

Que es un diccionario?

Estructura que relaciona claves y valores. Cada elemento es una dupla clave:valor. Tambien llamados arreglos asociativos (PHP), tablas de hash (Perl) o hashmaps (Java).

CLAVE
inmutable, unica
➔
VALOR
cualquier tipo
edades = {"Dante": 27, "Brenda": 18, "Malena": 23}
print(edades["Brenda"])  # 18

Reglas de los diccionarios

  • Las claves deben ser inmutables (numeros, strings, tuplas)
  • Los valores pueden ser de cualquier tipo (listas, dicts, etc.)
  • Las claves deben ser unicas
  • No son secuencias: no tienen orden, no usan subindice numerico
  • Las rebanadas no aplican
colores = {
    "Rojo":  [255, 0, 0],
    "Verde": [0, 255, 0],
    "Azul":  [0, 0, 255]
}

Agregar, modificar y eliminar

# Agregar o modificar (misma sintaxis)
colores["Gris"] = [128, 128, 128]  # crea nueva clave
colores["Rojo"] = [200, 0, 0]      # modifica existente

# Eliminar con del
del colores["Rojo"]

# Eliminar diccionario completo
del colores

Recorrer un diccionario

# for itera sobre las CLAVES
for color in colores:
    print(color, "=>", colores[color])

# items() devuelve tuplas (clave, valor)
for color, RGB in colores.items():
    print(color, "=>", RGB)

Metodos del diccionario

get(clave [, default])Devuelve valor o None (o default). Sin KeyError
keys()Secuencia de claves
values()Secuencia de valores
items()Secuencia de tuplas (clave, valor)
pop(clave [, default])Elimina y devuelve valor. KeyError si no existe y no hay default
dict.fromkeys(seq [, val])Crea dict desde secuencia con valor None o val
# get() seguro
a = colores.get("Cian", "No encontrado")

# keys(), values(), items()
claves = list(colores.keys())    # ['Rojo', 'Verde', 'Azul']
valores = list(colores.values())  # [[255,0,0], [0,255,0], ...]

# pop()
a = colores.pop("Verde")   # devuelve [0, 255, 0] y elimina

# fromkeys()
d = dict.fromkeys("aeiou", 0)
# {'a': 0, 'e': 0, 'i': 0, 'o': 0, 'u': 0}

# Diccionario por comprension
dic = {x: x**2 for x in range(1, 5)}
# {1: 1, 2: 4, 3: 9, 4: 16}

Acceso con clave inexistente = KeyError

edades["Carlos"]    # KeyError: 'Carlos'
edades.get("Carlos")  # None (sin error)
if "Carlos" in edades:  # verificar antes
