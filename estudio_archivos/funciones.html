def calcular_promedio(lista_notas):
    """Calcula el promedio de una lista de notas."""  ← docstring
    suma = sum(lista_notas)
    promedio = suma / len(lista_notas)
    return promedio  ← retorna el resultado

Reglas de nombres

  • Solo letras, numeros y guion bajo (_)
  • NO puede empezar con numero: 2calcular
  • NO usar palabras reservadas: def, return, if, for...
  • Nombres descriptivos con verbos: calcular_promedio, buscar_alumno, validar_nota
  • Usar snake_case: calcular_total (no calcularTotal)

Ubicacion en el programa

Las funciones van al PRINCIPIO del programa. El programa principal va DESPUES de la ultima funcion.

# PRIMERO: todas las funciones
def calcular_total(precio, cantidad):
    return precio * cantidad

def mostrar_resultado(total):
    print("Total:", total)

# DESPUES: programa principal
p = float(input("Precio: "))
c = int(input("Cantidad: "))
t = calcular_total(p, c)
mostrar_resultado(t)

Cada funcion hace UNA sola tarea

MAL Mezcla tareas

def calcular():
    n = int(input("Num: "))
    r = n ** 2
    print(r)
# Hace input + calculo + print!

BIEN Una tarea

def calcular_cuadrado(n):
    return n ** 2

# Programa principal
n = int(input("Num: "))
r = calcular_cuadrado(n)
print(r)

NO hacer input/print DENTRO de una funcion que hace otra tarea (calculo, busqueda, etc).

PROHIBIDO return dentro de un bucle

# MAL - NUNCA salir de una funcion desde adentro de un loop
def buscar(lista, valor):
    for elem in lista:
        if elem == valor:
            return True  # PROHIBIDO!
    return False

# BIEN - usar variable booleana
def buscar(lista, valor):
    encontrado = False
    for elem in lista:
        if elem == valor:
            encontrado = True
    return encontrado

Evitar funciones triviales

# MAL - funcion trivial que solo lee un dato
def leer_numero():
    return int(input("Numero: "))

# MAL - funcion trivial que solo suma
def sumar(a, b):
    return a + b

Si una funcion hace algo tan simple que se resuelve en UNA linea obvia, no tiene sentido crear una funcion para eso.

Variables y Alcance

Las variables son LOCALES

def mi_funcion():
    x = 10  # x es LOCAL a mi_funcion
    print(x)

mi_funcion()
print(x)  # ERROR! x no existe aca

Las variables del programa principal NO se pueden usar dentro de una funcion a menos que se pasen como parametro.

MAL Usar variables globales

total = 100  # variable global
def agregar(valor):
    return total + valor  # usa 'total' global - MAL!

# BIEN - pasar como parametro
def agregar(total, valor):
    return total + valor

Parametros

Formales vs Reales

Parametros Formales

Los que aparecen en la definicion (def).

def sumar(a, b):  # a, b = formales
    return a + b

Parametros Reales

Los que se pasan en la llamada.

resultado = sumar(5, 3)  # 5, 3 = reales

Mutables vs Inmutables

Inmutables (NO se modifican)

variables strings tuplas

def duplicar(x):
    x = x * 2  # modifica la copia LOCAL

n = 5
duplicar(n)
print(n)  # 5 (NO cambio!)

Mutables (SI se modifican)

listas sets diccionarios

def agregar(lista):
    lista.append(99)  # modifica el ORIGINAL

nums = [1, 2, 3]
agregar(nums)
print(nums)  # [1, 2, 3, 99] (SI cambio!)

Retorno de valores

Retorno multiple

def min_max(lista):
    return min(lista), max(lista)  # retorna tupla

minimo, maximo = min_max([4, 1, 7, 2])
print(minimo, maximo)  # 1 7

Valores por defecto

def potencia(base, exponente=2):
    return base ** exponente

print(potencia(3))     # 9  (usa exponente=2)
print(potencia(3, 3))  # 27 (usa exponente=3)

Parametros con nombre

def raiz(radicando, indice=2):
    return radicando ** (1/indice)

# Se pueden pasar en cualquier orden usando nombres:
print(raiz(indice=3, radicando=27))  # 3.0

Funciones Lambda

Funciones anonimas en una linea

# Funcion normal
def cuadrado(x):
    return x ** 2

# Equivalente con lambda
cuadrado = lambda x: x ** 2

# Mas ejemplos
doble = lambda x: x * 2
sumar = lambda a, b: a + b
es_par = lambda n: n % 2 == 0

import random
print(random.randint(1, 10))  # numero aleatorio entre 1 y 10

Resumen de Reglas Clave

1. NO hacer input/print dentro de funcion de calculo

2. NUNCA return dentro de un loop (for/while)

3. NO usar variables globales dentro de funciones

4. Funciones al PRINCIPIO, programa principal al FINAL

5. Cada funcion hace UNA sola tarea

6. NO crear funciones triviales

7. Pasar datos como PARAMETROS, no usar variables externas
