Concepto de Excepcion

Una excepcion es la indicacion de un problema ocurrido durante la ejecucion de un programa.

  • La "regla" es que los programas se ejecuten normalmente; la "excepcion" es que ocurra un problema.
  • El control de errores tradicional es preventivo: evita que el error ocurra (con if).
  • El control con excepciones es correctivo: intenta solucionar el problema despues de haber ocurrido.

Tradicional (preventivo)

if cantidad != 0:
    promedio = suma / cantidad
else:
    print("No se puede dividir por cero")

Excepciones (correctivo)

try:
    promedio = suma / cantidad
except ZeroDivisionError:
    print("No se puede dividir por cero")

Las excepciones permiten independizar la logica del programa del codigo de control de errores. Las aplicaciones son mas robustas y veloces.

Estructura try / except

El codigo ubicado entre try y except se denomina bloque protegido.

try:
    <codigo que puede provocar errores>
except <excepcion>:
    <codigo de tratamiento del error>

Multiples except

Se pueden escribir varios bloques except para distintos tipos de excepciones. Se analizan en orden.

try:
    a = int(input("Dividendo? "))
    b = int(input("Divisor? "))
    cociente = a // b
except ZeroDivisionError:
    print("No se puede dividir por cero")
except ValueError:
    print("Solo se permiten numeros enteros")
except:
    print("Error desconocido")  # ultimo recurso

Un except sin tipo captura CUALQUIER error, pero no se recomienda porque no permite diferenciarlos. Solo usarlo como ultimo recurso debajo de los demas.

Agrupar excepciones

Si varias excepciones reciben el mismo tratamiento, se agrupan entre parentesis:

except (ValueError, ZeroDivisionError):
    print("Datos invalidos")

Clausulas else y finally

Clausula else

  • Es opcional en un bloque try/except.
  • Se escribe despues de todos los except.
  • Solo se ejecuta si el try finalizo sin excepciones.
def leerentero(msj="Ingrese un entero: "):
    while True:
        try:
            n = int(input(msj))
        except ValueError:
            print("Dato invalido.")
            print("Intente nuevamente.")
        else:                # solo si NO hubo error
            break
    return n

Clausula finally

  • Es opcional, se escribe luego de try o de except.
  • Garantiza que una porcion de codigo se ejecute SIEMPRE, sin importar si hubo errores o no.
  • Se usa para liberar recursos: memoria, conexiones de red, archivos, sesiones de BD, etc.
  • Si se uso else, finally va despues de este.
try:
    <bloque protegido>
except <tipo de excepcion>:
    <codigo de control de errores>
else:
    <si no hubo error>          # opcional
finally:
    <codigo de saneamiento>     # SIEMPRE se ejecuta

Instruccion raise

Provocar excepciones con raise

  • raise se usa cuando se desea provocar una excepcion intencionalmente.
  • Puede ir seguida del nombre de la excepcion a producir.
  • Si no se detalla el tipo, se relanza la ultima excepcion producida.
def leerentero(msj="Ingrese un entero: "):
    while True:
        try:
            n = int(input(msj))
            break
        except ValueError:
            print("Dato invalido.")
            a = input("Desea ingresarlo otra vez? (S/N): ")
            if a.upper() == "N":
                raise  # relanza la excepcion
    return n

Instruccion assert

Afirmaciones con assert

  • assert incorpora un control expresado como una condicion.
  • Si la condicion es verdadera: no ocurre nada.
  • Si la condicion es falsa: se produce AssertionError.
  • Equivale a: if not (condicion): raise AssertionError
while True:
    try:
        mes = int(input("Ingrese el mes: "))
        assert 1 <= mes <= 12
        break
    except ValueError:
        print("Solo se permiten numeros.")
    except AssertionError:
        print("El mes debe estar entre 1 y 12.")
    print("Intente nuevamente.")

Se puede agregar un mensaje para diferenciar multiples assert:

assert len(palabra) > 5, "Palabra muy corta"
assert palabra.isalpha(), "Solo se permiten letras"

Observaciones Importantes

Errores de sintaxis NO se atrapan

Los errores de sintaxis son verificados durante el analisis sintactico del programa, no durante la ejecucion. No pueden ser capturados con try/except.

Excepciones no manejadas

Si ocurre un error que no coincide con ningun except, se traslada a bloques try/except exteriores. Si no se encuentra nada que la maneje, el programa se detiene con un mensaje de error.

Ejemplo completo: Funcion leerentero + division

def leerentero(msj="Ingrese un entero: "):
    while True:
        try:
            n = int(input(msj))
            break
        except ValueError:
            print("Dato invalido. Intente nuevamente.")
    return n

# Programa principal
while True:
    try:
        a = leerentero("Dividendo? ")
        b = leerentero("Divisor? ")
        cociente = a // b
        resto = a % b
        break
    except ZeroDivisionError:
        print("No se puede dividir por cero.")
print("Cociente:", cociente)
print("Resto:", resto)

Ejemplo: IndexError con finally en matriz

def imprimirmatriz(mat):
    filas = len(mat)
    columnas = len(mat[0])
    elementos = filas * columnas
    f = 0
    c = 0
    for i in range(elementos):
        try:
            print("%3d" %mat[f][c], end="")
        except IndexError:
            f = f + 1
            c = 0
            print()
            print("%3d" %mat[f][c], end="")
        finally:
            c = c + 1

Nota: finally se ejecuta en CADA iteracion, haya o no error. Se usa para avanzar la columna.
